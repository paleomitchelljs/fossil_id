# Fit harness — fitting analytical brachiopod parameters to GLB meshes

Five iterative rounds of critique → steelman → synthesis → revision built
this. Each round was scored against the same five GLBs:

  - Atrypa devoniana (atrypid, rounded biconvex)
  - Spinocyrtia iowensis (spiriferid, alate)
  - Pentamerus oblongus (pentamerid, elongate biconvex)
  - Mucrospirifer arkonensis (spiriferid, MUCRONATE alate)
  - Hebertella occidentalis (orthid, subcircular)

The five rounds, with the loss progression per species (lower = better,
chamfer + IoU sum across three views; chamfer term added in round 4 so
the absolute number is comparable only within rounds 1–3 and within 4–5):

| Species         | Rd1 (IoU) | Rd2 (IoU) | Rd3 (IoU) | Rd4 (CH+IoU) | Rd5 (CH+IoU) |
| --------------- | --------- | --------- | --------- | ------------ | ------------ |
| Atrypa devoniana    | 0.52 | 0.38 | 0.45 | 0.72 | 0.65 |
| Spinocyrtia iowensis| 0.54 | 0.50 | 0.48 | 0.92 | 0.98 |
| Pentamerus oblongus | 0.30 | 0.20 | 0.33 | 0.43 | 0.61 |
| Mucrospirifer       | 0.76 | 0.60 | 0.48 | 1.17 | 1.28 |
| Hebertella occ.     | 0.41 | 0.30 | 0.27 | 0.52 | 0.72 |

Pentamerus and Atrypa are the cleanest fits across the board.
Mucrospirifer is the worst — the mucronate wingtips need a different
outline parameterization than the piecewise super-ellipse.

## Pipeline

```
data/3d_models/*.glb
        │
        ▼
glb_loader.load_glb        → vertices, triangles
        │
        ▼
silhouette.canonicalise    → (X=lateral, Y=AP, Z=DV); AP normalised to 1
        │
        ▼
silhouette.mesh_silhouettes → (TOP, FRONT_anterior_cross_section, SIDE)
        │
        ▼
fit.fit_species            → model parameters minimising
                              (IoU + 4·chamfer) summed over 3 views
        │
        ▼
fit_round5.json + fit_round5.png    (per-species Params tuples + side-by-side)
```

## What works in the final model

- **Canonicalisation** is reliable: 5/5 species correctly oriented after
  the round-1 fix (use cross-section ASYMMETRY to disambiguate AP vs
  lateral axis rather than axis length, which fails on wide-alate shells).
- **TOP-view outline** is generated by an explicit closed curve
  (`top_outline`) — a piecewise super-ellipse with separate exponents
  for anterior (`p_ant`) and posterior (`p_post`) halves. The Y range
  is exactly [-0.5, +0.5], matching the canonical frame.
- **FRONT view** is a *cross-section at the anterior margin* (verts with
  y > 0.25), not a full lateral envelope. This is what real anterior
  photos show, and it's where the sulcus / fold are visible — the
  earlier full-envelope approach was dominated by the apex z and
  washed out the sulcus.
- **Loss** combines (1 − IoU) for area-overlap with chamfer distance
  for boundary-shape sensitivity. Chamfer is weighted 4× IoU.
- **Multi-restart optimisation** (4 restarts × random search + full
  coordinate descent, keeping the best across restarts).

## What the model can't do

- **Mucronate wingtips** (Mucrospirifer): the piecewise super-ellipse
  produces smooth tapered outlines. Real Mucrospirifer has sharply
  pointed wingtip extensions at the back. The model fits `lat_half=1.05`
  (the bound max) and `p_post=1.5` (the bound min), but still ends up
  with an oval-with-blunt-corners outline rather than the diamond
  with mucronate tips. The right fix is a different outline
  parameterization — maybe two extra "wing protrusion" parameters.
- **Sharp asymmetric beak prominence**: when the umbo curls over the
  hinge line in 3D (typical of mature dorsibiconvex shells), the
  canonical-coords assumption that umbo sits at y=-0.5 in the
  commissure plane breaks down. The fit ignores this — the model is
  purely a commissure-plane representation.
- **Resupinate / geniculate profiles**: the dome height function
  assumes a single positive apex (or single negative, if you flip
  signs per valve). Curvature-reversal isn't expressible.

## Usage

```bash
# Fit all 5 reference species and produce fit_round5.{png,json}
MPLCONFIGDIR=$TMPDIR/mpl ROUND_TAG=round5 python3 data/fit_harness/fit.py

# Re-render from saved JSON without refitting
MPLCONFIGDIR=$TMPDIR/mpl python3 data/fit_harness/render_results.py \
    data/fit_harness/fit_round5.json data/fit_harness/fit_round5_big.png

# Inspect a single mesh's silhouettes
MPLCONFIGDIR=$TMPDIR/mpl python3 data/fit_harness/silhouette.py \
    data/3d_models/brachiopod_atrypa_devoniana_pri_70763.glb
```

## Files

```
data/fit_harness/
  glb_loader.py        — minimal binary-glTF parser (no trimesh dep)
  silhouette.py        — canonical orientation + mesh silhouettes
  model.py             — analytical brachiopod (params, surface, outline)
  fit.py               — fitter (chamfer + IoU, multi-restart coord descent)
  render_results.py    — re-render comparison from a saved JSON
  fit_round{1..5}.png  — per-round visual progress
  fit_round{1..5}.json — fitted parameters per round
  fit_round5_big.png   — final, full-resolution comparison
  README.md            — this file
```

## Next steps (for whoever picks this up)

1. **Add mucronate wingtip parameters** — extend `top_outline` with a
   pair of wing-protrusion parameters (lateral extent + protrusion
   angle) that override the back-half super-ellipse near the cardinals.
2. **Port the fitted parameters into render.js** — wire `Params` →
   manifest → `svgTopView/Front/Side` so any taxon with a fitted
   tuple renders with its real shape.
3. **Re-fit at higher iteration budget** if anyone has cycles —
   coord-descent with 11-point golden-section line search and 4
   restarts × 100-eval random search is the bottleneck (~3.5 min
   per species; 5 species = 18 min). Could parallelise.
4. **Use the DAAL photogrammetry library** to fit more species —
   the harness will accept any GLB.
